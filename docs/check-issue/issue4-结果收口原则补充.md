# issue4 结果收口原则补充

日期：`2026-03-18`

主题：从结果发散回到结果收口

## 一、为什么要补这份说明

当前团队已经进入一个很典型的阶段：

1. 大家已经能把数据、特征、模型跑起来。
2. 也开始自发尝试更多模型方向，例如 `CNN`、`LSTM`。
3. 同时也有人已经开始考虑报告怎么写。

这说明项目推进是有效的，但也带来一个新风险：

如果没有统一“结果收口原则”，团队很容易重新进入各自探索、结果无法比较、最后难以统一主线的状态。

因此，这份补充文档的目标不是限制探索，而是统一：

1. 这次项目最终要讲什么
2. 哪些结果值得作为主线
3. 哪些结果更适合放到扩展探索

## 二、当前阶段最需要统一的认识

### 1. 这不是一个“做出华尔街级别预测模型”的项目

需要明确：

1. 股票市场是高噪声、多因素、动态博弈系统。
2. 仅凭一轮课程项目，不可能一步做出高稳定、高收益、强泛化的真实交易模型。
3. 如果对结果抱有“模型一上来就应当显著超过市场”的期待，判断标准会失真。

这次项目更合理的目标应该是：

建立一个规范、可复现、可解释的股票方向预测实验流程。

### 2. 这次项目真正重要的是边界意识

团队需要理解：

1. 哪些方法在当前任务下有效
2. 哪些方法在当前任务下无效
3. 为什么无效
4. 为什么看上去更复杂的模型，不一定比基线更好

课程项目里，这种“知道边界在哪里”的价值，往往比单次分数更重要。

### 3. 这次项目的关键贡献不是“神奇高分”，而是“清楚对比”

最后报告更值得回答的是：

1. 基础特征和扩展特征相比，是否真的带来了新增信息
2. 单股路线和组合路线，哪一种更容易形成稳定结论
3. 线性模型和树模型，在当前任务下各自有什么优缺点
4. 哪些模型只是看起来复杂，实际没有形成更好的可解释结果

## 三、接下来团队注意力应回到哪里

### 1. 回到特征工程

当前最值得投入的方向，不是继续无上限增加模型数量，而是回到特征本身：

1. 特征定义是否合理
2. 特征之间是否严重重叠
3. 某些特征为什么提供了更多信息
4. 某些特征为什么几乎不增加有效信息

建议把特征理解为两层：

1. 候选特征池
2. 核心特征集

这样做的好处是：

1. 方便控制实验复杂度
2. 方便判断性能变化到底来自模型，还是来自特征
3. 方便后续写报告时解释“为什么保留这些特征”

### 2. 回到评估指标

模型训练结果不能只看 `accuracy`。

还要同时看：

1. `precision`
2. `recall`
3. `f1`
4. `roc_auc`
5. 预测为正类的比例

原因很简单：

1. 有些模型虽然 `accuracy` 不低，但可能几乎总在预测上涨。
2. 有些模型虽然 `recall` 很高，但实际上区分能力很差。
3. 如果只看单一指标，容易误判模型是否真的学到了有效模式。

### 3. 回到可比较的实验协议

后续所有路线都应尽量在统一协议下比较：

1. 尽量使用一致的时间区间
2. 尽量使用一致的标签定义
3. 尽量使用一致的 train/test 切分方式
4. 尽量使用一致的结果输出格式

否则最后得到的不是“模型对比”，而只是“不同人各自做了不同实验”。

## 四、主线和扩展探索如何区分

### 1. 主线建议

当前项目的主线更适合锁定为：

1. 特征工程筛选与比较
2. `Logistic Regression` 基线
3. `XGBoost` 非线性基线
4. 单股路线与 pooled 路线的对比

原因：

1. 这更符合 `Machine Learning` 课程定位
2. 可解释性更强
3. 更容易形成稳定对比
4. 更适合在有限时间内完成结果收口

### 2. 扩展探索建议

`CNN`、`LSTM` 等方向可以继续尝试，但更适合作为：

1. 扩展探索
2. 补充讨论
3. 报告附录或“进一步工作”部分

不建议让这些方向反过来定义整个项目主线。

原因：

1. 这些模型对数据量、样本组织、时序窗口设计要求更高
2. 目前团队主问题不是模型复杂度不够，而是特征和评估还没有完全收紧
3. 如果主线过早转向深度学习，容易偏离课程重点，也容易导致比较失焦

## 五、issue4 最终应如何收口

`issue4` 的工作，不是单纯选一个最高分模型，而是完成以下几件事：

1. 判断哪些结果是有效结果
2. 判断哪些结果只是类别塌缩或偶然波动
3. 判断最终主线应落在单股还是 pooled
4. 判断最终主线应使用哪一组特征
5. 判断最终主线应使用哪个模型

最终推荐方案应同时满足：

1. 结果相对稳定
2. 解释逻辑清楚
3. 能和对照组形成比较
4. 容易进入报告和答辩表达

## 六、报告最值得讲的重点

最终报告建议重点讲清楚以下问题：

1. 为什么这个任务天然困难
2. 为什么基础价格量能特征的预测能力有限
3. 为什么特征工程比继续堆模型更关键
4. 为什么要同时看多个评估指标
5. 为什么某些模型看起来复杂，但并没有带来更好的主线价值
6. 为什么最终选择这个方案作为项目结论

## 七、一句话结论

这次项目最重要的，不是证明我们已经做出了强交易模型，而是证明我们已经建立了一个：

1. 可跑通
2. 可比较
3. 可解释
4. 可继续迭代

的股票预测实验框架。
