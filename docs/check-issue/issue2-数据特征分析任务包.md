# issue2：数据特征分析任务包

## 一、任务定位

`issue2` 不是“做点图看看”，而是项目后续所有建模工作的统一口径层。

它要回答的问题是：

1. 当前数据是否适合做 `next-day direction classification`？
2. 不同 ticker 的分布是否差异很大？
3. 统一特征口径应该怎么定？
4. 后续 `issue3A` 和 `issue3B` 应该共用哪些基础特征？

## 二、任务目标

完成后，团队应明确：

1. 标签定义不再反复变动。
2. 基础特征清单固定下来。
3. 单股和 pooled 建模都使用同一批基础字段。
4. 团队对数据分布、类别平衡、跨市场差异有共同认识。

## 三、输入

输入来源：

1. `data/raw/yahoo_daily_prices.csv`
2. `src/download_yahoo.py`
3. `src/check_raw_data.py`
4. `docs/data_source_plan.md`

## 四、建议产出

建议至少产出以下内容：

1. `EDA` notebook 或脚本
2. 数据分布说明
3. 标签定义说明
4. 基础特征清单
5. 一页结论说明

## 五、必须回答的问题

1. 各个 ticker 的时间范围是否一致？
2. 各 ticker 的涨跌比例是否明显偏斜？
3. 美国组与中概/港股组差异是否显著？
4. 哪些基础特征被纳入统一特征集？
5. 后续建模推荐：
- 单股
- pooled
- 两条线都保留

## 六、建议最少特征集

作为团队统一基线，建议至少包括：

1. `return_1d`
2. `return_5d`
3. `volume_change_1d`
4. `ma_gap_5`
5. `ma_gap_10`
6. `volatility_5d`

注意：  
`issue2` 可以提出扩展特征建议，但不要直接扩展到过多技术指标，避免 `issue3` 无法比较。

## 七、完成标准

`issue2` 完成，不是指“做完 notebook”，而是指：

1. 标签定义明确
2. 基础特征清单明确
3. 数据差异和风险说明明确
4. 下游建模能直接拿来用

## 八、协作接口

`issue2` 结束后，需要明确交给 `issue3A/3B` 的东西：

1. 最终标签定义
2. 最终基础特征清单
3. 数据切分建议
4. 一页 EDA 结论

## 九、风险提醒

1. 不要一边做 EDA 一边随意改任务定义。
2. 不要做出很多图，但没有结论。
3. 不要先扩展几十个特征，导致后续无法控制实验变量。
4. 不要把自己的观察只放在聊天里，不写成文档。
