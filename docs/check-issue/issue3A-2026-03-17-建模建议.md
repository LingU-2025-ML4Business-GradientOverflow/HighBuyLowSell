# issue3A 建模建议记录

日期：`2026-03-17`

主题：单股建模路线建议

## 一、总体判断

当前团队已经观察到：

1. 仅使用价格与成交量特征时，模型准确率大多在 `0.5` 左右。
2. 这个结果并不意外，也不等于项目没有价值。
3. 真正重要的是：如何把单股路线做成一条清晰、可解释、可比较的实验线。

因此，`issue3A` 的目标不应该是“继续盲目调参”，而应该是：

构建一条清晰的单股建模 baseline，并判断它是否值得进入最终模型选择阶段。

## 二、建议

### 1. 建议优先选择 1 到 2 个代表标的，而不是所有股票都深入做

原因：

1. 单股路线的价值在于结论清楚，而不是覆盖面越大越好。
2. 如果每只股票都做一版单独探索，时间不够，也不利于和 `issue3B` 对比。

其他重要信息：

建议优先：

1. `NVDA`
2. `GOOGL`

如需加一个中概对照，可补一个 `BABA` 或 `BIDU`。

### 2. 建议模型先固定为两类

原因：

1. `Logistic Regression` 可以作为简单、可解释的基线。
2. `XGBoost` 可以作为非线性树模型基线。
3. 这两个模型已经足够形成有效比较。

其他重要信息：

当前阶段不建议继续扩到更多模型，否则会增加比较噪声。

### 3. 建议不要只看 accuracy

原因：

1. 股价方向任务本身接近随机，accuracy 很容易停留在 `0.5~0.55`。
2. 有些模型虽然 accuracy 还可以，但可能几乎总在预测上涨或下跌。

其他重要信息：

建议至少同时看：

1. `accuracy`
2. `f1`
3. `roc_auc`
4. `prediction_positive_ratio`

### 4. 建议做“核心特征版”和“扩展特征版”的对比

原因：

1. 这样可以判断性能变化是来自模型，还是来自特征扩展。
2. 对报告写作也更有帮助，因为可以讲清楚“增加技术指标后是否有明显收益”。

其他重要信息：

如果时间不足，至少要写清楚当前最终使用的是哪组特征，以及选择理由。

### 5. 建议输出时保留完整实验信息

原因：

1. `issue4` 后续需要统一做模型选择。
2. 如果只留下“最优结果”，后面就无法判断结论是否稳健。

其他重要信息：

建议至少保留：

1. 使用的股票
2. 使用的特征组
3. 使用的模型
4. 时间切分方式
5. 指标表
6. 一页结论

## 三、单股路线当前最适合回答的问题

`issue3A` 应重点回答：

1. 单股是否比 pooled 更容易得到可解释结果？
2. 技术指标在单股场景下是否比 pooled 场景更有效？
3. 哪只代表标的更适合作为最终主线候选？

## 四、建议的实验收敛方式

建议把单股路线收敛成：

1. 单股 + 核心特征 + Logistic
2. 单股 + 核心特征 + XGBoost
3. 单股 + 扩展特征 + Logistic
4. 单股 + 扩展特征 + XGBoost

## 五、结论

`issue3A` 不是做“更多模型”，而是做“更清楚的单股技术判断”。  
如果这条线做得好，即使最终分数没有特别高，也能作为团队项目里非常有价值的一条解释性路线。
