# issue4：评估与模型选择任务包

## 一、任务定位

`issue4` 是项目的收敛层。

它的作用不是重新做一轮模型，而是：

1. 统一评价标准
2. 对比 `issue3A` 和 `issue3B` 的结果
3. 做最终模型选择
4. 确定最终报告技术主线

## 二、任务目标

必须回答以下问题：

1. 最终保留单股路线还是 pooled 路线？
2. 最终保留哪个模型？
3. 为什么选择这个模型，而不是另一个？
4. 如何用统一语言解释这个选择？

## 三、输入

输入应来自：

1. `issue2` 的统一标签与特征口径
2. `issue3A` 的单股路线结果
3. `issue3B` 的 pooled 路线结果

## 四、评价维度

建议统一从以下维度比较：

1. 分类指标
- accuracy
- f1
- roc_auc

2. 稳定性
- 是否依赖某一个标的
- 是否对不同市场组别差异很大

3. 策略意义
- 最小回测结果
- 是否优于简单基线
- 换手和回撤是否过高

4. 可解释性与报告可讲性
- 是否容易说明
- 是否适合课程答辩

## 五、建议产出

至少产出：

1. 最终模型对比表
2. 最终推荐模型
3. 推荐理由
4. 不选其他路线的理由
5. 一份可直接给 `issue5` 使用的总结

## 六、模型选择原则

最终模型不一定是“分数最高”的那个，而应综合考虑：

1. 分数是否稳定
2. 结果是否可解释
3. 路线是否清晰
4. 是否适合写进报告和答辩

## 七、完成标准

`issue4` 完成意味着：

1. 团队已经不再纠结主线选择
2. 最终模型已经确定
3. 技术叙事已经确定
4. `issue5` 可以直接开始写报告

## 八、风险提醒

1. 不要被单个高分结果误导。
2. 不要忽视回测层的意义。
3. 不要让评估阶段重新变成大规模探索阶段。
4. 不要等报告开始写了才决定用哪个模型。
