# issue3A：单股建模探索任务包

## 一、任务定位

`issue3A` 负责沿单股建模路线做探索。

这条路线要回答的问题是：

1. 单个标的是否比多标的联合建模更稳定？
2. 单股模型是否更容易解释？
3. 单股模型是否更适合作为课程作业的主线？

## 二、任务目标

目标不是“把所有模型都试一遍”，而是形成一个有判断力的单股基线。

建议重点：

1. 选择 1 到 2 个代表标的
2. 用统一基础特征做建模
3. 比较至少两种模型
4. 形成可以和 `issue3B` 对比的结论

## 三、建议代表标的

建议优先考虑：

1. `NVDA`
2. `GOOGL`
3. 如需中概对照，可补一个 `BABA` 或 `BIDU`

不要一上来把所有 ticker 都做成“单股精调”，那样时间不够，也不利于清晰比较。

## 四、建议模型范围

建议至少比较：

1. `Logistic Regression`
2. `XGBoost`

如有余力，可再加一个简单对照，但不建议无限扩模型。

## 五、输出要求

至少输出：

1. 单股实验代码
2. 指标表
3. 每个标的的结论
4. 哪个模型更适合单股场景的判断

## 六、必须回答的问题

1. 单股场景下，哪只股票更容易建模？
2. 单股场景下，`Logistic` 和 `XGBoost` 谁更稳？
3. 单股模型是否明显优于简单基线？
4. 单股路线是否值得保留到最终模型选择阶段？

## 七、完成标准

完成标准不是“模型跑通了”，而是：

1. 单股结果可复现
2. 模型对比可读
3. 结论清楚
4. 可以与 `issue3B` 做横向比较

## 八、协作接口

需要交给 `issue4` 的内容：

1. 单股实验结果表
2. 使用的特征清单
3. 训练/测试切分方式
4. 单股路线结论
- 值不值得保留
- 推荐保留哪个标的
- 推荐保留哪个模型

## 九、风险提醒

1. 不要因为是单股路线，就改掉 `issue2` 确定的特征口径。
2. 不要为了追分数不断加特征，导致无法说明提升来源。
3. 不要只交一个“最优结果”，要保留对比过程。
4. 不要把单股结果当成整体项目结论，必须留给 `issue4` 去统一判断。
